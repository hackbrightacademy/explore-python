Getting Familiar With Sublime

Even if you already have a text editor you know how to use, it’s required that you install and become familiar with Sublime Text. Sublime is the only editor installed on our lab computers, so you will be using it during pair programming.

If you haven’t used a programmer-specific text editor, you may wonder what’s so special about these over a program like TextEdit or Notepad. Programmer text editors contain lots of useful features that making writing code much easier. In particular, Sublime Text is one of the most popular editors used by programmers.

It’s free to download and use. It will suggest that you purchase a license to use it. You may either do so or you can click to continue the free trial. (The copies we use in lab are already fully licensed).

Download and install Sublime Text onto your laptop. You can find directions to help you accomplish this in the install specific to your operating system (Windows/Linux/OSX).

Practice making and saving a .py file and a .html file in Sublime Text. These are common file extensions that you will be using over the course of the class. You are probably familiar with other file extensions, such as .txt, .doc and .gif. When you save your file, you’ll want to include the file extension at the end of the name.

  • ex. myfile.py
  • ex. awesome.html

You’ll need to do this for your homework. It will also help with syntax, as Sublime color-codes syntax based on language when it knows what kind of file you’re editing. You must enter the extension while saving your file — it doesn’t default to a file type.

Here are a couple of great resources for you to read on using Sublime Text 2:

  • http://designmodo.com/sublime-text-2/ (page 1)
  • http://sublime-text-unofficial-documentation.readthedocs.org/en/sublime-text-2/basic_concepts.html

In order to familiarize yourself further with Sublime Text, try doing the following:

  • Save multiple files.
  • Open a folder instead of a file with the File > Open... utility.
  • Re-save (Save As...) a file that you’ve already saved with a different extension.
  • Play with the “Find” utility (Windows: Ctrl-F, OSX: Command-F) and try finding and replacing elements. (Look up how to do this if you’re unclear about how it might work.)

Opening the Vagrant Virtual Machine and Entering the Python Console

First, open a terminal.

In OS X, use the following Finder path:

Finder > Applications > Utilities > Terminal

Double-click to open. After a second, it should pop up with some information that looks similar to (but not exactly the same as):

Last login: Fri Apr 17 11:05:34 on ttys000
Balloonicorns-MacBook-Air:~ balloonicorn$

In Windows, go to:

Start menu > Applications > Git > Git Bash

Balloonicorn@LAPTOP MINGW32 ~
$

We will cover the information displayed later. For now, all you need to know is that you can type commands after the $ (everything up to and including the $ is known as the prompt).

In OS X or Windows, type the following commands in your terminal:

  • cd ~/vagrant
  • vagrant up
  • vagrant ssh

Note: if vagrant is not running on your machine, vagrant up may take several minutes to run (if vagrant is already running on your machine, vagrant up is superfluous but not harmful).

Once you are in the vagrant terminal, enter the command python to get to the interactive Python console (indicated by a >>> prompt):

vagrant@vagrant:~$ python
Python 2.7.6 (default, Jun 22 2015, 17:58:13)
[GCC 4.8.2] on linux2
Type "help", "copyright", "credits" or "license" for more information.
>>>

Running Python Files

At the vagrant prompt, you should be able to run your file by typing:

vagrant@vagrant:~$ cd src
vagrant@vagrant:~/src$ python yourlastname_info.py

(except, of course, fill in the last name that you specified earlier.)

The terminal should behave somewhat similiarly to the codecademy shell that you used to run your script online.

When your file is up and running, take a screenshot of the running terminal to send to us. Name the screenshot yourlastname_screenshot.png. Instructions for taking screenshots:

  • OS X

    Here’s a great guide for doing this on Mac:

    https://support.apple.com/en-us/HT201361

    I strongly suggest using the “Take a screenshot of part of your screen” directions. Taking a screenshot saves an image to your desktop.

  • Windows

    Use the key stroke combination WINDOW-PrtSc. (PrtSc is near the top right of keyboard.) The screen shot is saved to the folder “Pictures > Screenshots”.

    Alternatively, you can use the “Snipping Tool” instead.

Entering and Saving from the ipython Console

Similar to the interactive python console above, ipython allows you to type and immediately execute Python code. However, ipython provides the ability to save everything you type during the interactive session to a file (among many other useful features). Try it out! Type the following commands.

vagrant@vagrant:~$ ipython
Python 2.7.6 (default, Jun 22 2015, 17:58:13)
Type "copyright", "credits" or "license" for more information.

IPython 1.2.1 -- An enhanced Interactive Python.
?         -> Introduction and overview of IPython's features.
%quickref -> Quick reference.
help      -> Python's own help system.
object?   -> Details about 'object', use 'object??' for extra details.

In [1]: print "Hello, I'm in ipython!"
Hello, I'm in ipython!

In [2]: print "Let's try saving a file."
Let's try saving a file.

In [3]: save my_program 1-2
The following commands were written to file my_program.py:
print "Hello, I'm in ipython!"
print "Let's try saving a file."

What does this give you? Now the first and second line from your ipython session (line numbers are between the square brackets in the ipython prompt) have been saved to a python file called my_program.py.

You can run this file just like any other python file like the following:

vagrant@vagrant:~$ python my_program.py
Hello, I'm in ipython!
Let's try saving a file.

You can also add to it or edit it, like any other file, in Sublime Text.
