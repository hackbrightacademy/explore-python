OSX Setup

NOTE: These set up steps are all REQUIRED. If you run into trouble, please let us know ASAP, at ontrack@hackbrightacademy.com.

What Version of OS X Are You On?

You can figure out what version of OSX you’re on by clicking on the apple icon in the top left corner and going to About This Mac.

Supported versions:

  • 10.12: Sierra
  • 10.11: El Capitan
  • 10.10: Yosemite
  • 10.9: Mavericks

(If you have an earlier version, like Snow Leopard or Mountain Lion, you’ll need to upgrade, either your system or your entire computer. If you are using a laptop which came with OS X 10.8 or before, consider upgrading to a new computer – a more modern machine will not only run everything more smoothly, it will also mean less troubleshooting in the future.)

Setup Vagrant

Vagrant is a program which allows you to run a virtual Linux machine on your laptop. This virtual machine is where we’ll do much of our work.

Go to https://www.vagrantup.com/downloads.html and download the Mac version of Vagrant.

Open the .dmg file and run the Vagrant installer.

Open the terminal, and run the command pwd to make sure that you’re in your home directory.

$ pwd
/Users/YourUserName

(If it outputs /Users/YourUserName, then you are. If not, type cd ~, hit enter, and run pwd again.)

Now create a directory called vagrant (mkdir vagrant) and a directory called src (mkdir src). (You can run ls to list the files in your home directory, and you should see these two new directories in the list.) Now cd into your new vagrant directory (cd vagrant).

$ mkdir vagrant
$ mkdir src
$ ls
[lots of other files and directories]
src     vagrant
$ cd vagrant

Download this zip file, unzip it, and move the three files inside (Vagrantfile, hb-linux-setup.sh, and hb-vagrant-setup.sh) to the vagrant folder you created above.

Back in the terminal, run ls again, and you should see the three files you just moved.

$ ls
Vagrantfile     hb-linux-setup.sh     hb-vagrant-setup.sh

Now run the command vagrant up. This will download and install both VirtualBox (if you don’t already have it) and a copy of Ubuntu Linux. It will also start up the virtual Linux machine (though not yet give you a way to interact with said virtual machine). (Note that this process will probably take a few minutes. If it looks like it’s hung up, don’t panic. Get a cup of coffee and it should be done when you get back.)

$ vagrant up
Bringing machine 'default' up with 'virtualbox' provider...
[about a bajillion other lines of output - be patient and wait for your
prompt to reappear]
$

Errors during vagrant up?

If you run into errors during the vagrant up process, check out the “Vagrant Has Never Run Successfully on Your Machine” section of our Vagrant troubleshooting doc here: http://fellowship.hackbrightacademy.com/materials/resources/vagrant-troubleshooting/#vagrant-has-never-run-successfully-on-your-machine.

‘mesg: ttyname failed: Inappropriate ioctl for device’ Error?

You may see the above error twice during the vagrant up process, once at the beginning, and once at the end. The details are somewhat complicated, but the short version is that it’s not something to worry about.

To use the virtual machine, run vagrant ssh. You’ll notice that your prompt will change. Run the command ls, and you should see the src directory. Files in this directory will be mirrored to the src folder you created earlier on your Mac. Now cd into it (cd src).

$ vagrant ssh
Welcome to Ubuntu 16.04.1 LTS (GNU/Linux 4.4.0-31-generic x86_64)

  * Documentation:  https://help.ubuntu.com
  * Management:     https://landscape.canonical.com
  * Support:        https://ubuntu.com/advantage
vagrant@vagrant:~$ ls
src
vagrant@vagrant:~$ cd src
vagrant@vagrant:~/src$

To prove to yourself that these folders are actually linked, run the command touch testfile. If you run ls, you should now see testfile in your directory.

vagrant@vagrant:~/src$ touch testfile
vagrant@vagrant:~/src$ ls
testfile

Switch into Finder, open up the src folder, and testfile should be there as well. (Once you’ve seen that it is, you can trash it.)

Back in the terminal, run the command psql. Your prompt should change again, to vagrant=#.

vagrant@vagrant:~/src$ psql
psql (9.5.4)
Type "help" for help.

vagrant=#

Once you’ve verified that this works, take a screenshot (instructions here: https://support.apple.com/en-us/HT201361), and then use ctrl-D to get back to your virtual machine prompt. You will send in the screenshot later with your other homework from this assignment.

vagrant=# \q
vagrant@vagrant:~/src$

Congrats! Your virtual machine is now set up! To get back to your regular (Mac) terminal, hit ctrl-D again, and then run vagrant halt to turn off the virtual machine. The next time you want to start up the virtual machine, use vagrant up and vagrant ssh. (On a day-to-day basis once Hackbright starts, you’ll use vagrant ssh and ctrl-D to get into and out of the virtual machine, and only vagrant up and vagrant halt after you’ve restarted and before you shut down your computer. The machine will continue to run in the background, but it uses exceedingly little memory to do so.)

Sublime Text 2

Installing and Configuring Sublime

Sublime Text is a very popular editor for developers and is what we use on our pair programming workstations. Download a copy from http://www.sublimetext.com/2 and install it in your Applications folder.

To set up your Sublime like we have at the lab:

Press Command-, (comma) to open the Preferences.sublime-settings file. Copy/paste the following into that file:

{
  "translate_tabs_to_spaces": true,
  "tab_size": 4,
  "rulers": [80]
}
  • translate_tabs_to_spaces will automatically convert tab characters to spaces (this is required; without this, you won’t be successful at working with the code we give you)
  • tab_size will tell sublime that every tab should equal 4 spaces
  • rulers takes a list of integers at which to draw end-of-line markers in Sublime

Now save and close the settings file.

Additional Sublime Package: SublimeLinter

Sublime has a ‘linting’ plugin which provides unobtrusive hints as you type your code about syntax errors, style mistakes, and the like. In order to install it, you’ll first need to install a package manager plugin. Follow the instructions at the link below to install it (make sure you select Sublime 2):

https://sublime.wbond.net/installation#st2

Use the Package Manager you installed above to install the SublimeLinter plugin (go to Sublime Text 2 → Preferences → Package Control, then choose “Install Package” and enter “SublimeLinter”).
